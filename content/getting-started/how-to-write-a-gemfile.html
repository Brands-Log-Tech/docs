We support some default gems out of the box, but in order to use different versions or other gems you will need to include a Gemfile." tags = ["jekyll", "ruby", "gem", "gemfile"] title = "How to write a Gemfile" +++

Here is an example of a basic Gemfile.


source 'https://rubygems.org'
ruby "2.2.4"
gem 'jekyll', '1.5.1 '
gem 'jekyll-coffeescript'

Every Gemfile must have a source, in this case 

source 'https://rubygems.org'
 . This is where bundle (the program that installs all the gems in your Gemfile) will download the gems. Generally 
'https://rubygems.org'

is the only source you will need, since it is the official source for Ruby gems, however Forestry will support 3rd party sources as well.

You can specify the version of Ruby to use for your build as well. To do this simply add 

ruby "2.2.4"

to your Gemfile and change "2.2.4" to the version of Ruby you want to use. Forestry will build sites for ruby version 2.1.x and above, where x is the most recent bug fix for that version. You can find a complete list of ruby versions here. Forestry will support the versions listed under "Stable releases". If you don't specify a ruby version in your Gemfile, forestry will use the same minor version (the first two numbers in the version) as GitHub Pages, but we will use the latest patch (the last number in the version).

After this, you can start listing your gems. Optionally you can specify the version of the gem as the second argument to 

gem
. Any dependencies the gems have listed will be installed as well.
